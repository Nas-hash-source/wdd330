Chapter 7: Events

It adds interactions between the web page and the users.

                
                    document.body.addEventListener("click", doSomething); // when clicked, the doSomething function runs 
                    function doSomething(event){
                        console.log(event.target); //event.target refers to the node that fired the event.
                    }

Types of Events:
  • click: when a mouse button is clicked.
  • mousedown: when a mouse button is down.
  • mouseup: when a mouse button is up.
  • dblclick: when doubleclicked.
  • mouseover: when hovered
  • mouseout: after hovered, and back again when it moves away
  • keydown: when a key is pressed.
  • keyup: when the button is released.
  • event.key: the key pressed at that moment.
  • removeEventListener() // remove an event

event.preventDefault() is a method of the event object to stop the default behavior happening such as link and submit button.

Chapter 8: Forms

Forms

It's possible, and becoming more and more common, to process the information in a form on the front end before it is sent to the server.

Accessing Form elements

We access all of the forms in a HTML file using document.forms, so a specific form can be returned either through the index method, or using the name attribute document.forms.name. Be wary if you use accidently the name of the form as any property or methods of the document.forms object, use the bracket notation can be used to avoid this

form.elements returns all elements contained in the form.

form.nameIt can access a specific of the form controls using the name attribute

Form Properties and Methods

            
                form.submit() // submit the form automatically
                /* the element can be submitted manually by the user employing a button or 
                input element with a type attribute of submit, or even an input element with a type attribute of image */
                form.reset() // reset all the form controls back to their initial values
                form.action ='/an/other.url' // to send to a different URL to be processed on the server

Form Events

  • the focus event is enabled when the cursor is placed inside the element.
  • the blur event: when the user moves the focus away from the form element.
  • the change event: when the user moves the focus awau after changing it.
  • the submit event: when the user submits a form.

Submitting a Form

We can use JavaScript to intercept the form before sending to the server. Using the event.preventDefault() hold the default action of the submit button 

            
                input.value //the value of the input, and can be assigned to another value

Form Controls

The defautl type of an input field is text, the type="text" makes it explicit.
The type="password": hidden to be read on screen.
Checkbox: true if checked, false if unchecked.
Radio type: only one can be checked
Hidden Input type: just hidden, not secret
File Input fields: to upload files

The select Drop-Down List: form.city.selectedIndex : returns the index of the option selected.
form.city.options[0].text : access the text of the options in the drop-down list using the index.

Form Validation

The form validation should be applied both on the client-side and server-side of the program to make it more sanitized.

Chapter 9: The Window Object

BOM

It contains information about the browser and computer screen

Going Global

Global variables can be created without using var, elt or const.

parseInt(), isNaN() and some functions are all window object.

Dialogs

alert(), confirm() and prompt() are not part of ECMScript standard.

            
                window.alert('Hello'); //pause the execution and display a message in a display box
                window.confirm(); // stop the execution, if ok: return true, if cancel: return false
                window.prompt(); // allows the use to enter text, return as a string

Browser information:

            
                window.navigator.userAgent; // return the browser and os information
                //it can be modified though, and becoming deprecated
                window.location; // the URL location of the page
                window.location.href; // the href
                window.location.protocol;
                window.location.hostname;
                window.location.port;
                window.location.pathname;

Browser History:

            
                window.history.go(1); // goes forward 1 page or we can use window.history.forward()
                window.history.go(0); // reloads the current page
                window.history.go(-1); // goes back 1 page or we can use window.history.back()

Controlling Window

            
                const popup = window.open('https://sitepoint.com',
                'SitePoint', 'width=400, height=400, resizable=yes');
                //first parameter: URL, second parameter: name of the window, third parameter: attributes
                popup.close(); // close the window
                window.moveTo(x, y) // move the position of the window
                window.resizeTo(width, height); // resize the window

Can be used to popup ad and annoying stuff but there are alternative ways to avoid that.

Cookies

Text file that contains a list of name/value pairs seperated by semicolons

The EU directive recommended to ask for permission before setting cookies

            
                document.cookie = 'name=Superman'; // add cookie without overwriting

By default, cookies are deleted when the session is finished, but you can add expiry dates

            
                const expiryDate = new Date();
                const tomorrow = expiryDate.getTime() + 1000 * 60 * 60 * 24;
                expiryDate.setTime(tomorrow);
                document.cookie = `name=Batman; expires=${
                expiryDate.toUTCString()}`; // add an expiry date

By default, it's only read inside the same directory and domain as the file was set.

            
                document.cookie = 'name=Batman; path=/'; //any page in the root directory can read the cookie;
                document.cookie = 'name=Batman; domain=sitepoint.com'; // will allow subdomains too to read the cookie;

Secure cookies

document.cookie = 'name=Batman; secure' // it's only transmitted over a secure HTTPS network

Deleting a cookie can be done by setting an expiration in the past.

Many developers use Cookies.js or jsCookie for dealing with cookies due to the JavaScript's cookie basic handling.

Timing Functions

window.setTimeout() has a callback function as a first parameter and a number of milliseconds as its second parameter.

window.clearTimeout() prevents it to happen.

            
                window.setInterval() //same as the window.setTimeout() but it repeats the action depending on the second parameter
                window.clearInterval(summon) // clear the interval

Chapter 12: Object-Oriented Programming in JavaScript

Encapsulation

This principle is applied when you keep the logic hidden by the outside world

Polymorphism

Various object has the same method but different functionalities

Inheritance

Taking the features of one object to a new object. It inherits the properties or the methods.

Constructor Functions

            
                const Dice = function(sides=6){
                    this.sides = sides;
                    this.roll = function() {
                    return Math.floor(this.sides * Math.random() + 1)
                    }
                    } // constructor function, the keyword this represent object within

The class declaration syntax was introduced in ES6, it has a function named constructor () {}

Static Methods: it is called by the class directly rather than by instances of the class.

Prototypal Inheritance: property and methods of the class's prototype can be accessed by every object in that class.

Chapter 15: Modern JavaScript Development

Organization is needed when it comes to bigger project when there will be thousands of codes.

Libraries

They can be used to complete common tasks without repetitive code

jQuery is the most popular of all the JavaScript libraries. Its best advantage is that it supports the older browsers.

Underscore and Lodash: very similar libraries. Using one of them in your project can give access to a large number of well-tested utility functions that may be required in projects.

Advantage: used by lots of people and help you avoid reinvent the wheel.

Disadvantage: performance issue, but sometimes small and optimized.

Modular JavaScript

It is way of organizing codes with modules to make the project more maintainable.

            
                export const PI = 3.1415926; // export the variable
                import { PI } from './pi.js'; // import the variable to be used
                export {variance, mean} // export functions from a module
                import { mean, variance } from './stats.js'; // import selectively the functions
                import * as stats from './stats.js'; // import all from a module
                export default square; // export a single function or variable, usually used for modules and functions
                import square from './square.js'; // import a default export
                import sq from './square.js'; // use an alias from the default export

Node.js Modules

            
                module.exports = x => x * x; // export in node js
                const square = require('./squareFunction'); // then import it
